    About Ridda.ai

    Ridda.ai is the web interface for PokeMenu (digital collectible set registry and launchpad) and PokeBro NFT (fixed-cap collection, max 100000). PokeMenu lets the owner create sets with a name hash, max per set, and price; opening a sale allows users to mint from that set by paying in ETH. Minted tokens are PokeBro NFTs. Fees go to treasury; creator and launchpad wallet receive shares. Deploy PokeBro first, then PokeMenu; set PokeMenu as PokeBro minter and set PokeBro address in PokeMenu.

    PokeMenu: PMU_MAX_SETS 64, PMU_POKEBRO_CAP 100000, PMU_MAX_MINT_PER_TX 24. PokeBro: PBRO_MAX_SUPPLY 100000. Not financial advice.

    How it works

    • Sets: Owner creates a set with createSet(nameHash, maxPerSet, priceWei). Each set gets an ID. Owner can open/close sale and update price.
    • Mint: When a set sale is open, call mintFromSet(setId, count) with msg.value = priceWei * count. PokeMenu mints PokeBro NFTs to you (token IDs from nextTokenId). Fee goes to treasury; half of the remainder to set creator, half to launchpad wallet.
    • PokeBro: ERC721-style NFT, max supply 100000. Only the minter (PokeMenu) can mint. Owner can change minter to another address if needed.

    Deployment order

    1) Deploy PokeBro (constructor has no args; minter is deployer). 2) Deploy PokeMenu (constructor has no args; treasury, vault, launchpad wallet are set internally). 3) Call PokeBro.setMinter(pokeMenuAddress). 4) Call PokeMenu.setPokeBroNft(pokeBroAddress). 5) Create sets and open sales as needed.

    FAQ

    What is PokeMenu?
    PokeMenu is a launchpad contract that manages collectible sets. Each set has a price and max supply. When the owner opens a sale, users can mint from that set by paying in ETH; they receive PokeBro NFTs.
    What is PokeBro?
    PokeBro is an NFT contract with a fixed max supply of 100000. Only the minter (typically PokeMenu) can mint. You can transfer and trade PokeBro NFTs like any ERC721.
    Where does my payment go?
    A fee (configurable BPS) goes to the PokeMenu treasury. The rest is split 50/50 between the set creator and the launchpad wallet.
    Why do I need two contract addresses?
    PokeMenu is the launchpad (sets + mint logic); PokeBro is the NFT contract. You set both so the UI can read sets from PokeMenu and your balance from PokeBro.
    Can I mint if sale is closed?
    No. Only sets with sale open accept mintFromSet. The Sets tab shows which sets are open.
    What is the max supply?
    PokeBro has a global cap of 100000 tokens. PokeMenu will not mint beyond that. Each set has its own max per set (set by owner).

    Tips

    Always verify contract addresses on-chain (e.g. block explorer) before connecting and minting. This UI does not custody any funds; all transactions are signed in your wallet. If a set shows "Sale open", you can mint; required ETH is (set price × count). PokeMenu and PokeBro are separate contracts: deploy both, wire minter and NFT address as described in Deployment order. For large collections, "My PokeBro NFTs" may scan up to 200 token IDs; for full enumeration use a subgraph or indexer.

    Security & best practices

    Use a hardware or dedicated wallet for minting. Confirm the PokeMenu and PokeBro contract addresses on the block explorer (same chain you are on). Do not share your seed phrase or private key with this or any site. Check that the set sale is open and remaining supply is sufficient before sending a transaction. If the platform is paused, mintFromSet will revert. After deployment, ensure PokeBro minter is set to PokeMenu and PokeMenu has the PokeBro NFT address set so mints succeed.

    Glossary

    Set
    A collectible run on PokeMenu with a name hash, max supply per set, price in wei, and creator. Identified by set ID.
    Launchpad
    The PokeMenu mechanism: open a set for sale so users can mint by paying ETH.
    PokeBro NFT
    ERC721-style NFT with max supply 100000; minted by PokeMenu when users mint from a set.
    Treasury
    Immutable address on PokeMenu that receives the fee portion of each mint.
    Vault
    Immutable address on PokeMenu (e.g. for protocol reserves).
    Launchpad wallet
    Immutable address on PokeMenu that receives half of the (post-fee) mint payment; the other half goes to the set creator.
    Name hash
    Bytes32 identifier for a set (e.g. keccak256 of the set name). Stored on-chain.
    BPS
    Basis points; 10000 BPS = 100%. Fee is (payment × feeBps) / 10000.

    Quick start (users)

    1. Connect your wallet (e.g. MetaMask) via the Connect button.
    2. Enter the PokeMenu contract address in the first input and the PokeBro contract address in the second. Get these from the project or block explorer.
    3. Open the Sets tab to see available sets and which have sale open.
    4. Open the Mint from set tab, choose a set ID and count (1–24). The required ETH is shown.
    5. Click Mint and confirm the transaction in your wallet. You will receive PokeBro NFTs.
    6. Open My PokeBro NFTs to see tokens you own (scan may be limited to 200).

    Troubleshooting

    • Mint fails with "sale not open" — The set sale is closed. Only the contract owner can open it.
    • Mint fails with "exceeds set supply" — The set has reached its max per set. Choose a lower count or another set.
    • Mint fails with "exceeds global supply" — PokeBro has reached 100000 minted. No more mints possible.
    • Mint fails with "insufficient payment" — Send at least (set price × count) wei. Check the required ETH display.
    • Mint fails with "PokeBro not set" — Owner has not set the PokeBro NFT address in PokeMenu. Contact deployer.
    • Mint fails with "platform paused" — Owner has paused the platform. Wait or contact deployer.
    • My NFTs empty but I minted — Ensure the PokeBro address is correct and you are on the right chain. Try Refresh.
    • Sets tab empty — Ensure PokeMenu address is correct and the contract has at least one set created.

    Chain and network

    Ridda.ai works with any EVM chain where PokeMenu and PokeBro are deployed. Connect your wallet and ensure you are on the same network as the contracts. Switch network in your wallet if needed. Contract addresses are chain-specific; do not reuse mainnet addresses on testnets or vice versa.

    Version and support

    This interface is provided as-is. Smart contract logic is determined by PokeMenu and PokeBro deployments; Ridda.ai only reads and sends transactions. For contract bugs or upgrade paths, refer to the contract owner or deployment documentation. UI version: 1.0.0.

    PokeMenu owner flows

    After deployment, owner calls setPokeBroNft(pokeBroAddress). Create a set with createSet(nameHash, maxPerSet, priceWei); nameHash is typically keccak256(abi.encodePacked("Set Name")). Open the sale with openSale(setId). To stop sales, closeSale(setId). To change price, updateSetPrice(setId, newPriceWei). Optionally setPlatformPaused(true) for emergency stop, or setFeeBps(bps) to change the treasury fee (max 400 BPS). Sweep functions (sweepTreasury, sweepVault, sweepLaunchpad) send contract balance to the respective immutable addresses; use if ETH accumulates.

    PokeBro owner flows

    Deploy PokeBro; minter is deployer by default. Call setMinter(pokeMenuAddress) so PokeMenu can mint. Only one minter at a time. If you need to rotate minter (e.g. new launchpad), call setMinter(newAddress). Token IDs are assigned sequentially from 0; PokeMenu tracks nextTokenId and mints that ID. Transfers and approvals follow ERC721 semantics; use ownerOf(tokenId), balanceOf(account), transferFrom, approve, setApprovalForAll as needed for marketplaces.

    Summary

    Ridda.ai connects your wallet to PokeMenu (launchpad) and PokeBro (NFT). You can view sets, mint when sale is open, and view your PokeBro balance. All state is on-chain; this site does not store data. Keep your contract addresses saved; you need both to use the full flow. For deployers: deploy PokeBro then PokeMenu, set minter and NFT address, create sets, open sales. For users: connect wallet, enter addresses, mint from open sets, check My PokeBro NFTs.

    Technical details

    • PokeMenu uses ReentrancyGuard and Pausable (OpenZeppelin). Constructor sets treasury, vault, launchpadWallet to fixed addresses; no constructor args.
    • PokeBro has no external dependencies; custom ERC721-style implementation. name "PokeBro NFT", symbol "PBRO".
    • Mint flow: user sends ETH to mintFromSet; contract computes fee and creator/launchpad shares, forwards ETH, then calls IPokeBroNft(pokeBroNft).mint(msg.sender, nextTokenId) in a loop; nextTokenId increments.
    • Set IDs start at 1; setCounter is the current number of sets. getSetIds() returns an array of all set IDs.
    • tokenIdToSetId mapping on PokeMenu records which set each token ID came from (for display or indexing).
    • Snapshot events are emitted when mints occur; setSnapshots and _setSnapshotIds allow querying history per set.
    • feeBps is in basis points; max 400 (PMU_MAX_FEE_BPS). Default 85. Owner can change via setFeeBps.
    • Platform pause (setPlatformPaused) stops mintFromSet; owner and OpenZeppelin pause() are separate (Pausable).

    Checklist: deployer

    1. Deploy PokeBro contract (no args). Note the deployed address.
    2. Deploy PokeMenu contract (no args). Note the deployed address.
    3. Call PokeBro.setMinter(pokeMenuAddress) from the PokeBro owner account.
    4. Call PokeMenu.setPokeBroNft(pokeBroAddress) from the PokeMenu owner account.
    5. Create one or more sets: PokeMenu.createSet(keccak256("Set Name"), maxPerSet, priceWei).
    6. Open a set for sale: PokeMenu.openSale(setId).
    7. Share PokeMenu and PokeBro contract addresses with users (e.g. for Ridda.ai).
    8. Optional: adjust fee with setFeeBps(bps), or pause with setPlatformPaused(true).

    Checklist: user

    1. Install or use a Web3 wallet (e.g. MetaMask) and ensure you have ETH for gas and mint cost.
    2. Open Ridda.ai (or the deployed frontend).
    3. Click Connect and approve the wallet connection.
    4. Enter the PokeMenu contract address in the first input field.
    5. Enter the PokeBro contract address in the second input field.
    6. Open the Sets tab and confirm at least one set shows "Sale open".
    7. Open the Mint from set tab; enter the set ID (e.g. 1) and count (1–24).
    8. Confirm the required ETH shown matches (set price × count). Send that amount when you click Mint.
    9. Sign the transaction in your wallet. After confirmation, PokeBro NFTs will be in your wallet.
    10. Open My PokeBro NFTs to see your token IDs (or check your wallet / block explorer).

    Notes

    Name hash for a set is typically keccak256(abi.encodePacked("Human Readable Name")) in Solidity; in JS you can use ethers.keccak256(ethers.toUtf8Bytes("Name")). The same set ID is used across the UI and contract. Token IDs are globally sequential (0 to 99999); they are not per-set. If you integrate a marketplace, use the PokeBro contract address and standard ERC721 approval/transfer flows. Ridda.ai does not handle metadata or IPFS; that is outside the scope of the launchpad and NFT contracts described here.

    Constants: PMU_BPS_BASE 10000, PMU_SET_SALT and PMU_LAUNCHPAD_DOMAIN for domain separation. PokeBro PBRO_DOMAIN and PBRO_CHAIN_SALT. Immutable addresses in PokeMenu: treasury, vault, launchpadWallet (set in constructor; verify on deployment).

    API summary

    PokeMenu: getFrontendConfig(), getSetIds(), getSetInfo(setId), getSetPrice(setId), mintFromSet(setId, count) payable, canMintFromSet(setId, count). Owner: createSet, openSale, closeSale, updateSetPrice, setPokeBroNft, setPlatformPaused, setFeeBps, sweepTreasury, sweepVault, sweepLaunchpad. PokeBro: balanceOf(account), totalSupply(), ownerOf(tokenId), approve, setApprovalForAll, transferFrom, safeTransferFrom. Owner: setMinter. Minter: mint(to, tokenId).

    Reference: set info fields

    getSetInfo(setId) returns: nameHash (bytes32), maxPerSet (uint256), priceWei (uint256), creator (address), mintedFromSet (uint256), saleOpen (bool), createdAtBlock (uint256). Use these to display set details and compute remaining supply as maxPerSet - mintedFromSet.
